%%--------------------------------------------
Write a C function, called {\tt vvincr}, that takes as input
\be
\item \(x\), type is (\(M_1, n\), {\tt float})
\item \(y\), type is (\(M_1, n\), {\tt float})
\item \(n\), type is (\(M_0\), {\tt int})
\ee
and updates \(x\) as follows
\be
\item \(x_i \leftarrow x_i + y_i\)
\ee

  \be
\item Produce highly optimized C code.
\item Use AVX2 intriniscs wherever possible.
\item Do {\bf not} assume that all pointers, \(x, y\), are aligned on 64-byte boundaries
\item Assume that there is no overlap in the memory accessed using \(x, y\)
\item Do {\bf not} assume that \(n\) is a multiple of 
  (64 / {\tt sizeof(float)})
  \ee
